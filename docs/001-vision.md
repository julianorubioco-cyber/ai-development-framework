# Visão do AI Development Framework

## Propósito

O AI Development Framework (ADF) organiza agentes de IA para desenvolver software
com mais previsibilidade, menos retrabalho e menor consumo desnecessário de contexto.

O framework transforma uma solicitação em um fluxo verificável:

```text
entender → delimitar → planejar → aprovar → implementar → testar → revisar → memorizar
```

## Problema resolvido

Agentes de desenvolvimento frequentemente:

- começam a alterar código antes de entender a tarefa;
- leem arquivos demais;
- repetem análises já realizadas;
- perdem decisões entre sessões;
- declaram conclusão sem evidência;
- misturam memória de projetos diferentes.

O ADF cria contratos claros para evitar esses comportamentos.

## Público

- desenvolvedores individuais;
- pequenas equipes;
- agências;
- consultorias;
- empresas que usam agentes de IA em projetos de software.

## Resultado esperado

Uma pessoa deve conseguir abrir um projeto, executar `/implement` e receber:

1. entendimento explícito;
2. plano proporcional ao risco;
3. aprovação antes de alterações;
4. implementação controlada;
5. revisão posterior baseada em evidências;
6. memória curada somente dentro do projeto.

## Limites

O ADF não substitui:

- revisão humana;
- controle de acesso;
- backups;
- testes reais;
- políticas de segurança;
- responsabilidade técnica;
- ambientes de homologação e produção.

## Norte do projeto

O framework deve permanecer:

- agnóstico à tecnologia;
- simples de instalar;
- explícito sobre riscos;
- econômico em contexto;
- auditável;
- modular;
- compatível com evolução incremental.
