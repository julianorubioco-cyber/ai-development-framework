# Orquestração

`/implement` é o comando principal.

## Antes de agir

1. Entender a solicitação.
2. Identificar o contexto mínimo.
3. Classificar tamanho e risco.
4. Criar ou atualizar especificação.
5. Criar plano quando necessário.
6. Executar preflight.
7. Mostrar entendimento, escopo, riscos e estratégia.
8. Aguardar aprovação explícita.

## Depois de agir

1. Implementar.
2. Executar testes relevantes.
3. Revisar cada critério por evidência.
4. Encaminhar problemas à Skill adequada.
5. Repetir a revisão, no máximo oito ciclos.
6. Validar release quando aplicável.
7. Atualizar memória curada.

Mudança de escopo, exclusão de dados, migração destrutiva, custo externo ou troca
de tecnologia exigem nova aprovação.
