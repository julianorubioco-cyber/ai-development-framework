# Contribuindo

1. Crie uma branch.
2. Mantenha cada Skill com uma única responsabilidade.
3. Não inclua dados de clientes, chaves ou segredos.
4. Preserve compatibilidade com projetos de stacks diferentes.
5. Atualize a documentação e o changelog.
6. Teste a instalação em uma pasta temporária antes de abrir um pull request.
